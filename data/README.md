# 데이터 설명

## 개요

본 프로젝트는 **AI hub**의 "경구약제조합 5000종" 데이터셋을 기반으로 약 이미지에서 실시간 클래스 검출 모델을 개발합니다.

스마트폰으로 촬영한 알약 이미지에서 최대 4개의 알약을 탐지하고, 각 알약의 클래스(종류)와 위치(바운딩 박스)를 예측합니다.

---

## 데이터 소스

### 원천 데이터 (TS*2*조합.zip)

- **출처**: AI hub - Training > 원천데이터 > 경구약제조합 5000종
- **구성**: 다양한 조명 환경과 배경에서 촬영한 알약 이미지
- **포함**: 원본 이미지 파일들

### 라벨링 데이터 (TL*2*조합.zip)

- **출처**: AI hub - Training > 라벨링데이터 > 경구약제조합 5000종
- **구성**: COCO-style JSON 형식의 어노테이션 파일
- **포함**: 바운딩 박스 좌표, 클래스 라벨, 이미지 메타데이터

---

## 구조 및 내용

### `original/` 폴더

전처리 전, 데이터가 저장되는 공간입니다.

```
original/
├── images/          # 원본 알약 이미지들
├── labels/          # COCO JSON 어노테이션 파일들
└── train_annotations/  # 훈련용 JSON 어노테이션
```

### `yolo_dataset/` 폴더

**최종 사용 데이터셋** - YOLO 모델 학습을 위해 변환된 포맷입니다.

```
yolo_dataset/
├── images/    # 이미지 (jpg/png)
└── labels/    # YOLO txt 포맷 어노테이션
```

**YOLO 포맷 설명:**

- 각 이미지에 대응하는 `.txt` 파일 존재
- 포맷: `<클래스_ID> <x_center> <y_center> <width> <height>` (정규화 좌표)
- 예시: `2 0.45 0.52 0.3 0.4` (클래스 2, 중심 좌표 0.45/0.52, 너비/높이 0.3/0.4)

### `새로만든json/` & `수정json/` 폴더

추가로 생성되거나 수정된 JSON 샘플 파일들입니다.

---

## 데이터 전처리 파이프라인

1. **원본 데이터 로드**: `original/` 폴더에서 이미지와 JSON 어노테이션 읽기
2. **어노테이션 정제**: 불완전하거나 이상한 데이터 제거/수정
3. **YOLO 포맷 변환**: COCO JSON → YOLO txt 형식
4. **Train/Val/Test 분할**: 8 : 2로 분할
5. **최종 데이터셋**: `yolo_dataset/` 폴더에 저장

전처리 코드: `src/data_engineer/` 참조

---

## 주의사항

- **원본 파일**: `original/` 폴더는 용량이 크므로 필요시에만 접근하세요.
- **YOLO 확장자**: 이미지는 jpg/png, 어노테이션은 txt 형식입니다.
- **클래스 매핑**: 각 클래스 ID는 `data.yaml` 파일에 정의되어 있습니다.
- **좌표 정규화**: YOLO 포맷의 좌표는 0~1 범위로 정규화되어 있습니다.
